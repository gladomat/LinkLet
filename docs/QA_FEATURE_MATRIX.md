# LinkLet — Canonical QA Feature & Test Matrix

> Single source of truth for feature discovery, test scenarios, execution status, and defects.
> Generated by the continuous QA agent. Keep this file authoritative; update on every iteration.
>
> **Columns:** Feature ID · Feature Name · User Story · Expected Behaviour · Edge Cases · Test Cases · Status · Defects · Severity · Notes · Last Tested
>
> **Column mapping** (the 11 required fields → table headers): Feature ID=`ID` · Feature Name=`Feature` · User Story=`User Story` · Expected Behaviour=`Expected Behaviour` · Edge Cases=`Edge Cases` · Test Cases=`Test Cases` · Current Status=`Status` · Defect Count=`Def` · Severity=`Sev` · Notes=`Notes` (inline where present; per-defect notes live in §0 Defect Register) · Last Tested Date=`Last Tested`. Every feature row carries all 11.
>
> **Status legend:** `DISC` discovered · `TESTED` automated coverage exists & passing · `PARTIAL` partial coverage · `UNTESTED` no automated coverage · `WAIVED` coverage intentionally deferred (rationale in Notes) · `DEFECT` open defect.
> **Severity:** Critical / High / Medium / Low / — (none).
>
> VIEW-05 (share) and VIEW-14 (expand/collapse) now have real Robolectric Compose tests (`NoteActionSheetsTests`) — no longer waived.

App: Android (Kotlin, Jetpack Compose, MVVM + Hilt, Room, WorkManager). Org-roam note viewer/editor with 2-pass SQLite index and WebDAV/Nextcloud sync. 8 navigation routes.

> **Phase 2 test scenarios:** the `Test Cases` column below is the one-line summary. The full per-feature scenario inventory — every feature × {Happy, Error, Boundary, Invalid, Security, Performance, Mobile} — lives in the companion [`QA_TEST_PLAN.md`](./QA_TEST_PLAN.md) (85/85 features, covering test class named where automated coverage exists).
